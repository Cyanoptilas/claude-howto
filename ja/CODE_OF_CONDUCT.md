# 貢献者行動規範

## 私たちの約束

私たちは、すべての人に対して歓迎的で刺激的なコミュニティを提供することを約束します。年齢、体型、カースト、障害、民族、性的特徴、性自認および性表現、経験レベル、学歴、社会経済的地位、国籍、外見、人種、宗教、または性的指向に関わらず、コミュニティへの参加がハラスメントのない体験であることを誓います。

すべての貢献者が尊重され、大切にされ、最良の仕事ができるよう力を与えられると感じられる、ポジティブで包括的かつ安全な環境を提供することに取り組んでいます。

## 私たちの基準

ポジティブな環境を作るために貢献する行動の例を以下に示します。

### 敬意を持つ
- 歓迎的で包括的な言葉を使う
- 異なる意見、視点、経験を尊重する
- 建設的な批判を素直に受け入れる
- 文化的・言語的差異を尊重する
- 多様なバックグラウンドを認め、称える

### 協力的であること
- 対立を解決するために協力する
- 他者の貢献に対してクレジットを与え、認める
- 他者の学習と成長を助ける
- 知識を共有し、質問する
- コミュニティメンバーをサポートする

### プロフェッショナルであること
- 議論を集中的かつ生産的に進める
- 軽蔑的または侮辱的な発言を避ける
- Issue や議論でトピックに集中する
- 明確で敬意ある表現を使う
- 他者のメッセージには善意があると仮定する

### 包括的であること
- 新しい貢献者を歓迎する
- プロジェクトに新しく参加した人のオンボーディングを助ける
- 建設的なフィードバックを提供する
- 異なる視点を持つ人々を含める
- 静かな声が聞こえる場を作る

## 許容されない行動

以下の行動はハラスメントとみなされ、私たちのコミュニティでは許容されません。

### ハラスメントと差別
- 性別、性自認および性表現、性的指向、障害、精神疾患、神経学的非定型性、外見、体型、年齢、人種、または宗教に関する侮辱的なコメント
- 個人のライフスタイルの選択や実践に関する歓迎されないコメント
- 意図的な誤ったジェンダー表記、または拒否された名前の使用
- 不必要またはトピック外のわいせつな画像や行動
- 同意なしの、またはやめるよう求めた後の身体的接触および擬似的身体的接触

### 虐待的な行動
- 他者に向けた暴力の脅迫または暴力的な言葉
- 個人への暴力または暴力的な言葉の扇動
- 意図的な威圧
- ストーキングや追跡
- 繰り返しの接触によるハラスメント
- 不適切とみなされるその他の行為

### オンラインハラスメント
- オンラインディスカッションの持続的な妨害
- 他者の個人情報の公開（ドックス行為）
- 歓迎されない性的関心または誘い
- トローリングや意図的な妨害
- スパムまたは繰り返しの広告
- 誤情報またはデマの拡散

### 報復
- 違反を報告した人に対する報復行為
- 許容されない行動を報告した人への不当な扱い
- 報復としての報告の脅し

## 適用範囲

この行動規範は以下に適用されます。
- すべてのコミュニティスペース（GitHub の Issue、プルリクエスト、ディスカッションなど）
- プロジェクトのリポジトリおよびドキュメント
- プロジェクトが主催または代表するイベントや集まり
- プロジェクトやコミュニティを代表する公共の場
- プロジェクト業務に関わるプライベートなコミュニケーション

この行動規範はすべてのプロジェクトスペースに適用され、個人がプロジェクトやコミュニティを公共の場で代表する場合にも適用されます。プロジェクトやコミュニティの代表例としては、公式プロジェクトのメールアドレスの使用、公式ソーシャルメディアアカウントへの投稿、オンラインまたはオフラインのイベントでの任命された代表としての行動などが挙げられます。

## 執行

コミュニティのリーダーは、許容できる行動の基準を明確にし執行する責任があり、不適切・脅迫的・攻撃的・有害とみなすあらゆる行動に対して適切かつ公正な是正措置を取ります。

### 違反の報告

許容されない行動を経験または目撃した場合、またはその他の懸念がある場合は、以下の方法で報告してください。

1. **メール**: プロジェクトメンテナーに連絡する（連絡方法が利用可能な場合）
2. **GitHub Issues**: プライベート報告機能（GitHub 設定で利用可能な場合）
3. **ダイレクトメッセージ**: プロジェクトメンテナーに直接連絡する

以下の情報を提供してください。
- あなたの連絡先情報
- 関係者の名前（共有していただける場合）
- 事件の説明
- 状況および関連するリンク
- その事件があなたにどのような影響を与えたか

**すべての報告は機密として扱われます。**

### 調査と対応

コミュニティのリーダーは以下を行います。
- すべての苦情を迅速かつ公平にレビューし調査する
- 報告者に関して守秘義務を維持する
- 報告のステータスについて報告者に回答する
- 調査結果に基づいて適切な措置を取る

考えられる措置：
- 非公開での警告またはコーチング
- 公開謝罪（違反者が同意した場合）
- コミュニティスペースからの一時的または永久的な排除
- 貢献者権限の剥奪
- 必要に応じて GitHub またはその他のプラットフォームへのエスカレーション

### 異議申し立てプロセス

決定に誤りがあると思われる場合は、以下の方法で異議申し立てができます。
1. 新しい情報や状況を提供する
2. 複数のメンテナーによるレビューを要求する
3. 最初の決定から30日以内にフォローアップする

## 建設的な意見の相違のためのガイドライン

意見の相違は技術コミュニティでは自然なことです。敬意を持って意見を異にする方法を以下に示します。

### すること
- ✅ アイデアに焦点を当て、個人を攻撃しない
- ✅ 明確化のための質問をする
- ✅ 反対意見の有効な点を認める
- ✅「私」を主語にした表現を使う（「私が反対する理由は...」）
- ✅ 出典と証拠を示す
- ✅ 相手の視点に感謝する

### しないこと
- ❌ 個人攻撃をする
- ❌ 軽蔑的な言葉を使う（「それは馬鹿げている」）
- ❌ 悪意があると仮定する
- ❌ 関係のない過去の対立を持ち出す
- ❌ トーンをエスカレートさせたり敵対的になったりする
- ❌ 意図的に他者の意見を歪めて伝える

### 建設的な意見の相違の例
```
❌ 悪い例: 「それはくだらないアイデアで、プロジェクトを全然理解していない。」

✅ 良い例: 「おっしゃる点は理解できますが、このアプローチは大規模なプロジェクトには
スケールしないかもしれないと懸念しています。このベンチマークに示されている
パフォーマンスへの影響についてはどうお考えですか？ぜひご意見をお聞きしたいです。」
```

## 橋を架けること

私たちは対話と和解の力を信じています。

- 誰かがあなたの行動を指摘したら、耳を傾けて振り返る
- 害を与えた場合は謝罪する意志を持つ
- 他者の学習と成長をサポートする
- 人は間違いを犯すことを認識する
- 責任追及ではなく解決策に焦点を当てる
- 人々に責任を持ってもらいながら善意を仮定する

## この規範を守ることのメリット

敬意を持ち包括的なコミュニティを維持することで、私たちは以下を実現できます。
- 多様な視点を持つ多様な貢献者を引き付ける
- 協力によってより良い解決策を生み出す
- より強固な関係を構築する
- プロジェクトを新参者にとってより歓迎的にする
- より高品質なドキュメントと例を作成する
- 学習と成長の文化を育む

## 特別な考慮事項

### メンテナー向け
- この規範の遵守において手本を示す
- 報告に対して迅速に対応する
- 決定について透明性を保つ
- 公平に対立を解決する支援をする
- すべての人に対して安全な環境を作る

### 貢献者向け
- メンテナーや他の貢献者に善意があると仮定する
- 建設的なフィードバックを提供する
- 新参者をサポートする
- コミュニティ基準の施行を助ける
- コミュニティ文化に対してオーナーシップを持つ

### 全員向け
- すべてのメッセージの背後には人間がいることを忘れない
- フラストレーションを感じたら休憩を取る
- サポートが必要な場合はコミュニティリーダーに連絡する
- 自分が参加したいと思うコミュニティを作る手助けをする
