## 概要

Claude Code の上級機能は、プランニング・推論・自動化・制御メカニズムでコア機能を拡張します。これらの機能により、複雑な開発タスク・コードレビュー・自動化・マルチセッション管理のための高度なワークフローが実現します。

**主な上級機能：**
- **プランニングモード** — コーディング前に詳細な実装プランを作成
- **拡張思考** — 複雑な問題への深い推論（`Alt+T` / `Option+T` で切り替え）
- **自動モード** — バックグラウンドの安全分類器が各アクションを実行前にレビュー（リサーチプレビュー）
- **バックグラウンドタスク** — 会話をブロックせずに長い処理を実行
- **パーミッションモード** — Claude が実行できることを制御（`default`・`acceptEdits`・`plan`・`auto`・`dontAsk`・`bypassPermissions`）
- **ヘッドレスモード** — 自動化と CI/CD 向けに非インタラクティブで実行（`claude -p`）
- **セッション管理** — 複数の作業セッションを管理

---

## プランニングモード

プランニングモードを使うと、Claude は実装前に複雑なタスクを検討し、レビューして承認できる詳細なプランを作成します。

### プランニングモードの使い方

**スラッシュコマンド：**
```bash
/plan ユーザー認証システムを実装
```

**設定ファイルで自動有効化：**
```json
{
  "permissions": {
    "defaultMode": "plan"
  }
}
```

### いつ使うか

✅ 以下の場合に使用：
- 複雑なマルチファイルリファクタリング
- 新機能の実装
- アーキテクチャの変更
- データベースマイグレーション
- 主要な API の再設計

❌ 以下の場合は不要：
- 単純なバグ修正
- フォーマットの変更
- 単一ファイルの編集
- クイッククエリ

## 拡張思考

拡張思考は Claude が回答を提供する前により深く推論できるようにします。複雑な問題・設計上の決定・マルチステップの推論に最適です。

### 有効化方法

```bash
# キーボードショートカット
Alt+T (Windows/Linux) または Option+T (macOS)

# または /effort コマンドで設定
/effort high
/effort max  # Opus 4.7 が必要
```

---

## 自動モード

自動モードは、各アクションを実行前にバックグラウンドの安全分類器がレビューする機能です（リサーチプレビュー）。

```bash
# Max サブスクライバー（Opus 4.7）は --enable-auto-mode フラグ不要（v2.1.112以降）
claude --enable-auto-mode
```

---

## バックグラウンドタスク

バックグラウンドタスクを使うと、現在の会話をブロックせずに長い処理を実行できます。

```bash
# バックグラウンドタスクを開始
/tasks

# Claude は長い処理を自動的にバックグラウンドで実行できる
"この大規模なリファクタリングをバックグラウンドで実行して"
```

---

## パーミッションモード

| モード | 説明 |
|------|-------------|
| `default` | すべての操作でパーミッションを確認 |
| `acceptEdits` | ファイル編集を自動承認（bash コマンドは確認） |
| `plan` | 変更を提案するが実行はしない |
| `auto` | バックグラウンド分類器がレビュー（リサーチプレビュー） |
| `dontAsk` | ほとんどの操作のパーミッションをスキップ |
| `bypassPermissions` | すべての確認をスキップ（自動化に使用） |

```json
// ~/.claude/settings.json
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
```

## ヘッドレスモード

ヘッドレスモード（プリントモード）では、Claude Code を非インタラクティブで実行できます。CI/CD・スクリプト・自動化に最適です。

```bash
# 基本的な使用方法
claude -p "テストを実行してレポートを生成して"

# ファイルのコンテンツを処理
cat error.log | claude -p "このエラーを説明して"

# JSON 出力
claude -p --output-format json "関数をリスト"

# バッチ処理
for file in src/*.js; do
  claude -p "$(cat $file) をレビューして" > "review_$(basename $file).md"
done
```

## セッション管理

```bash
# セッションを再開
/resume
/resume auth-feature

# セッションの名前を変更
/rename my-session-name

# セッションを分岐
/branch

# CLI でのセッション管理
claude -c                      # 最近のセッションを続ける
claude -r "session-name"       # 名前で再開
claude --from-pr 42            # PR にリンクされたセッションを再開
```

## インタラクティブ機能

### キーボードショートカット

| ショートカット | アクション |
|------------|--------|
| `Enter` | メッセージを送信 |
| `Shift+Enter` または `Option+Enter` | 改行を挿入 |
| `↑` / `↓` | コマンド履歴を移動 |
| `Ctrl+C` | 現在の応答を中断 |
| `Esc` + `Esc` | チェックポイントインターフェースを開く |
| `Alt+T` / `Option+T` | 拡張思考を切り替え |
| `/` | 利用可能なコマンドを表示 |

## 音声ディクテーション

プッシュトゥトーク音声入力を使って Claude Code に話しかけることができます。20言語の音声認識をサポートしています。

```bash
# 音声ディクテーションを切り替え
/voice
```

## Git ワークツリー

Git ワークツリーを使って並行作業のための独立したブランチを作成できます。

```bash
# ワークツリーモードで起動
claude -w

# 新しいワークツリーを作成して作業
# 複数の独立したタスクを並行して実行
```

サンドボックスモードは OS レベルのファイルシステムとネットワークの分離を提供します。

```bash
# サンドボックスを切り替え
/sandbox
```

## 設定

### settings.json の構造

```json
{
  "permissions": {
    "defaultMode": "default",
    "allow": ["Bash(git *)"],
    "deny": []
  },
  "hooks": {
    "SessionStart": [...]
  },
  "model": "claude-sonnet-4-6",
  "autoMemoryDirectory": "/path/to/memory"
}
```

### 設定ファイルの場所

| 場所 | スコープ |
|----------|-------|
| `~/.claude/settings.json` | ユーザー設定（全プロジェクト） |
| `.claude/settings.json` | プロジェクト設定（コミット済み） |
| `.claude/settings.local.json` | ローカルオーバーライド（git 無視） |

## ベストプラクティス

- 複雑な作業にはプランニングモードを使う
- 繰り返しの編集には `acceptEdits` モードを設定して確認を減らす
- セッションに説明的な名前を付けて管理を簡単にする
- バックグラウンドタスクで長い処理を並行して実行する
- CI/CD パイプラインにはヘッドレスモードを使う
