# プランニングモードの例

Claude Code でプランニングモードを効果的に使用する実際の例。

## 例1: REST API の構築

### プランニングモードなし

```
ユーザー: ブログ用の REST API を作って

Claude: API を作成します...
[すぐにコーディングを開始、要件を見落とす可能性がある]
```

### プランニングモードあり

```
ユーザー: /plan ブログ用の REST API を作って

Claude: ブログ API の包括的なプランを作成します。

## ブログ REST API 実装プラン

### フェーズ1: プロジェクトセットアップ（15分）
1. Express を使った Node.js プロジェクトを初期化
2. TypeScript 設定をセットアップ
3. 依存関係をインストール（express、pg、jwt、joi）
4. 環境変数を設定
5. プロジェクト構造をセットアップ

### フェーズ2: データベースレイヤー（30分）
6. データベーススキーマを設計
   - users テーブル（id、email、password_hash、created_at）
   - posts テーブル（id、user_id、title、content、published、created_at、updated_at）
   - comments テーブル（id、post_id、user_id、content、created_at）
7. マイグレーションファイルを作成
8. データベースコネクションプールをセットアップ
9. データベースモデルを実装

### フェーズ3: 認証（45分）
10. ユーザー登録エンドポイントを実装
11. JWT を使ったログインエンドポイントを実装
12. 認証ミドルウェアを作成
13. bcrypt でパスワードハッシュを追加
14. トークンリフレッシュロジックを実装

### フェーズ4: Posts API（40分）
15. GET /api/posts — 公開投稿一覧（ページネーション付き）
16. GET /api/posts/:id — 単一投稿取得
17. POST /api/posts — 新規投稿作成（認証必須）
18. PUT /api/posts/:id — 投稿更新（認証必須、所有権チェック）
19. DELETE /api/posts/:id — 投稿削除（認証必須、所有権チェック）
20. GET /api/users/:id/posts — ユーザーの投稿取得

### フェーズ5: Comments API（30分）
21. GET /api/posts/:id/comments — 投稿のコメント取得
22. POST /api/posts/:id/comments — コメント追加（認証必須）
23. DELETE /api/comments/:id — コメント削除（認証必須、所有権チェック）

ユーザー: このプランを承認します。実装を進めてください。

Claude: プランに従って実装を開始します...
[各フェーズを順番に実装]
```

## 例2: 複雑なリファクタリング

```
ユーザー: /plan モノリスをマイクロサービスに分解して

Claude: モノリス分解の包括的なプランを作成します。

## マイクロサービス移行プラン

### 現状分析
- 現在のモノリスのサービス境界を特定
- 依存関係マップを作成
- データ所有権を分析

### フェーズ1: ドメイン分離（2週間）
1. 境界付けられたコンテキストを特定
2. ドメインオブジェクトを整理
3. サービス間通信パターンを定義

### フェーズ2: データ分離（3週間）
4. 各サービス用のデータベースを分離
5. データ同期戦略を実装
6. マイグレーションスクリプトを作成

### フェーズ3: サービス抽出（4週間）
7. 認証サービスを抽出
8. ユーザーサービスを抽出
9. 注文サービスを抽出
10. API ゲートウェイを実装

ユーザー: プランを承認します。フェーズ1から始めてください。
```

## プランニングモードのベストプラクティス

- 複雑な実装では常にプランを確認してから承認する
- プランの各フェーズを小さく実行可能なステップに分ける
- 不明な点や前提条件はプラン段階で質問する
- 承認前にプランをチームメンバーとレビューする
