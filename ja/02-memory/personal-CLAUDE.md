# 私の開発設定

## 自己紹介
- **経験レベル**: フルスタック開発 8年
- **得意な言語**: TypeScript, Python
- **コミュニケーションスタイル**: 直接的で例示付き
- **学習スタイル**: コード付きビジュアルダイアグラム

## コードの好み

### エラーハンドリング
try-catch ブロックと意味のあるエラーメッセージによる明示的なエラー処理を好みます。
汎用的なエラーは避け、デバッグのために常にエラーをログに記録してください。

### コメント
コメントは「何を」ではなく「なぜ」を説明するものにしてください。コードは自己文書化されているべきです。
コメントはビジネスロジックや自明でない判断を説明するものであるべきです。

### テスト
TDD（テスト駆動開発）を好みます。
まずテストを書いてから実装する。
実装の詳細ではなく動作にフォーカスする。

### アーキテクチャ
モジュール的で疎結合な設計を好みます。
テスタビリティのために依存性注入を使う。
関心事を分離する（Controllers、Services、Repositories）。

## デバッグの好み
- プレフィックス付きの console.log を使用: `[DEBUG]`
- コンテキストを含める: 関数名、関連変数
- 可能な場合はスタックトレースを使用
- ログに常にタイムスタンプを含める

## コミュニケーション
- 複雑な概念はダイアグラムで説明する
- 理論の前に具体的な例を示す
- 変更前後のコードスニペットを含める
- 最後に要点をまとめる
