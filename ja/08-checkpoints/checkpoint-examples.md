# チェックポイントの例

Claude Code でチェックポイントを効果的に使用する実際の例。

注意: チェックポイントはすべてのユーザープロンプトで自動的に作成されます。手動で保存する必要はありません。巻き戻すには `Esc` を2回（`Esc+Esc`）押すか、`/rewind` を使ってチェックポイントブラウザを開きます。

## 例1: データベースマイグレーション

### シナリオ
MySQL から PostgreSQL へのマイグレーションを直接マイグレーションとデュアルライトアプローチの両方で試したい。

### ワークフロー

```
ユーザー: MySQL から PostgreSQL への直接マイグレーションを実装して
      [チェックポイント A が自動作成]

Claude: 直接マイグレーションを実装します...
[データベースドライバーを変更、クエリを更新、スキーマを修正]

ユーザー: テストを実行して
      [チェックポイント B が自動作成]

Claude: テスト実行中...
❌ 15個のテストが失敗 - コネクションプールの問題

ユーザー: このアプローチには問題がある。デュアルライトパターンを試そう。
[Esc+Esc を押し、チェックポイント A を選択、「コードと会話を復元」を選択]

ユーザー: 両方のデータベースに書き込むデュアルライトパターンを実装して

Claude: デュアルライトマイグレーションを実装します...
[デュアルライトレイヤーと段階的なマイグレーション戦略を実装]

ユーザー: テストを実行して

Claude: テスト実行中...
✅ すべてのテストが通過

ユーザー: デュアルライトアプローチの方が明らかに安全。コミットしよう。
```

## 例2: パフォーマンス最適化

### シナリオ
複数のパフォーマンス最適化戦略を試す。

### ワークフロー

```
ユーザー: API レスポンスタイムを最適化して。現在: 450ms
      [チェックポイント「ベースライン」が自動作成]

Claude: データベースクエリキャッシュを追加します...
[Redis キャッシュを実装]

ユーザー: パフォーマンステストを実行して

Claude: 結果: 平均レスポンスタイム 280ms（38%改善）

ユーザー: 別のアプローチを試そう。
[Esc+Esc を押し、「ベースライン」チェックポイントを選択、「コードと会話を復元」を選択]

ユーザー: 代わりにデータベースクエリ最適化を試して

Claude: データベースクエリを最適化します...
[インデックスを追加、N+1クエリを最適化]

ユーザー: パフォーマンステストを実行して

Claude: 結果: 平均レスポンスタイム 180ms（60%改善）

ユーザー: クエリキャッシュと組み合わせてみよう
[チェックポイントに戻り、両方のアプローチを組み合わせる]

Claude: 最適化されたクエリに Redis キャッシュを追加します...

ユーザー: パフォーマンステストを実行して

Claude: 結果: 平均レスポンスタイム 95ms（79%改善）🎉

ユーザー: 完璧！コミットしよう。
```

## 例3: デバッグセッション

### シナリオ
複数の仮説でトリッキーなバグを追う。

### ワークフロー

```
ユーザー: アプリケーションにメモリリークがある
      [チェックポイント「デバッグ前」が自動作成]

Claude: 調査します...
[コードを分析]

ユーザー: イベントリスナーの問題かもしれない。修正して。

Claude: [イベントリスナーのクリーンアップを修正]

ユーザー: メモリプロファイラーを実行して

Claude: まだメモリリークが続いている - イベントリスナーは原因ではなかった

ユーザー: 戻って別の仮説を試そう。
[Esc+Esc を押し、「デバッグ前」チェックポイントを選択]

ユーザー: データベース接続かもしれない？

Claude: [コネクションプーリングの修正を実装]

ユーザー: メモリプロファイラーを実行して

Claude: まだリークしている - 接続も原因ではなかった

ユーザー: 別の角度から試そう。
[Esc+Esc で戻る]

ユーザー: 循環参照をチェックして

Claude: [キャッシュレイヤーの循環参照を発見・修正]

ユーザー: メモリプロファイラーを実行して

Claude: ✅ メモリリーク修正！キャッシュレイヤーの循環参照が原因だった。

ユーザー: この修正をコミットしよう。
```

## 主なポイント

1. **チェックポイントは自動的** — すべてのユーザープロンプトでチェックポイントが作成される。手動保存は不要
2. **Esc+Esc または /rewind を使用** — チェックポイントブラウザへのアクセス方法
3. **適切な復元オプションを選ぶ** — コード・会話・両方・要約のどれかを必要に応じて選択
4. **実験を恐れない** — チェックポイントで大胆な変更を安全に試せる
5. **git と組み合わせる** — 探索にはチェックポイント、確定した作業には git を使う
6. **長いセッションを要約する** — 「ここから要約」で会話を管理しやすくする
