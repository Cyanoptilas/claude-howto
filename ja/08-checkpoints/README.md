# チェックポイントと巻き戻し

チェックポイントを使うと、Claude Code セッションで会話の状態を保存して以前の時点に戻ることができます。さまざまなアプローチの探索・ミスからの回復・代替ソリューションの比較に非常に役立ちます。

## 概要

チェックポイントは会話の状態のスナップショットで、以下を含みます：
- 交わしたすべてのメッセージ
- 加えたファイルの変更
- ツール使用履歴
- セッションコンテキスト

## 主要概念

| 概念 | 説明 |
|---------|-------------|
| **チェックポイント** | メッセージ・ファイル・コンテキストを含む会話状態のスナップショット |
| **巻き戻し** | 以前のチェックポイントに戻り、その後の変更を破棄する |
| **分岐点** | 複数のアプローチを探索するためのチェックポイント |

## チェックポイントへのアクセス方法

### キーボードショートカット
`Esc` を2回（`Esc` + `Esc`）押すとチェックポイントインターフェースが開き、保存済みのチェックポイントを参照できます。

### スラッシュコマンド
`/rewind` コマンド（別名：`/checkpoint`）を使用：

```bash
/rewind
# または
/checkpoint
```

## 巻き戻しオプション

巻き戻し時に5つのオプションが表示されます：

1. **コードと会話を復元** — ファイルとメッセージの両方をそのチェックポイントに戻す
2. **会話を復元** — メッセージのみを巻き戻し、現在のコードはそのまま保持
3. **コードを復元** — ファイルの変更のみを戻し、完全な会話履歴を保持
4. **ここから要約** — この時点以降の会話を AI 生成の要約に圧縮し、コンテキストウィンドウの空きを作る
5. **キャンセル** — 現在の状態に戻る

## 自動チェックポイント

Claude Code は以下のタイミングで自動的にチェックポイントを作成します：
- すべてのユーザープロンプトの後
- 重要なファイル変更の後
- ツール実行の後

## ユースケース

### 異なるアプローチを試す

```markdown
ユーザー: このアルゴリズムをリファクタリングしてください

Claude: [アプローチ A を実装]

# 気に入らない場合は /rewind で元に戻り、別のアプローチを試せる
```

### 実験的な変更からの回復

```markdown
# 大胆な変更を試みた後：
/rewind
# → 「コードと会話を復元」を選択して安全な状態に戻る
```

### A/B テスト

```markdown
# チェックポイント A から実装 X を試す
# /rewind で戻って実装 Y を試す
# 両方の結果を比較する
```

## チェックポイントのベストプラクティス

| すべきこと | すべきでないこと |
|------|---------|
| 実験的な変更の前にチェックポイントを確認 | チェックポイントなしで本番コードに大きな変更を加える |
| 複数のアプローチを探索するために自由に使う | コードのバックアップとして唯一の手段にする |
| 「要約」オプションで長いセッションを管理 | 重要な作業のバージョン管理の代わりにする |
