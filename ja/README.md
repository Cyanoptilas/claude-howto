## 問題点

Claude Code をインストールして、いくつかプロンプトを試した。それで次は？

- **公式ドキュメントは機能を説明するだけで、組み合わせ方を教えてくれない。** スラッシュコマンドの存在は知っているけれど、フック・メモリ・サブエージェントと連携させて実際に時間を節約するワークフローを構築する方法は分からない。
- **明確な学習パスがない。** MCP とフックはどちらを先に学ぶべき？スキルとサブエージェントは？結局すべてをさらっと読んで、何も身につかない。
- **例が基本的すぎる。** 「Hello World」のスラッシュコマンドでは、メモリを使い専門エージェントに委任しセキュリティスキャンを自動実行する本番用コードレビューパイプラインの構築方法は学べない。

Claude Code のパワーの90%を使えていない — そして何を知らないかさえ分かっていない。

## Claude How To が解決すること

これは単なる機能リファレンスではありません。Claude Code のすべての機能を実際に使えるように、**体系的・ビジュアル・事例主導**で教えるガイドです。今日から自分のプロジェクトにコピーして使える実践的なテンプレートが揃っています。

| | 公式ドキュメント | このガイド |
|--|---------------|------------|
| **形式** | リファレンスドキュメント | Mermaid ダイアグラム付きビジュアルチュートリアル |
| **深さ** | 機能の説明 | 内部動作の仕組み |
| **例** | 基本的なスニペット | すぐに使える本番環境対応テンプレート |
| **構成** | 機能別 | 段階的な学習パス（初級→上級） |
| **オンボーディング** | 自己誘導 | 時間の目安付きガイドロードマップ |
| **自己評価** | なし | ギャップを発見しパーソナライズドパスを構築するインタラクティブクイズ |

### 得られるもの

- **チュートリアルモジュール10本** — スラッシュコマンドからカスタムエージェントチームまで、Claude Code のすべての機能を網羅
- **コピペ可能な設定** — スラッシュコマンド・CLAUDE.md テンプレート・フックスクリプト・MCP 設定・サブエージェント定義・完全なプラグインバンドル
- **Mermaid ダイアグラム** — 各機能の内部動作を図示。「どうやるか」だけでなく「なぜそうなるか」が理解できる
- **ガイド付き学習パス** — 11〜13時間で初心者からパワーユーザーへ
- **内蔵の自己評価** — Claude Code で `/self-assessment` や `/lesson-quiz hooks` を直接実行してギャップを特定

## 使い方

### 1. 自分のレベルを確認する

[自己評価クイズ](LEARNING-ROADMAP.md#レベルを確認する)を受けるか、Claude Code で `/self-assessment` を実行する。すでに知っていることに基づいてパーソナライズドロードマップを取得。

### 2. ガイド付きパスを進む

10モジュールを順番に進める — 各モジュールは前のモジュールの上に積み重なる。学習しながらテンプレートを自分のプロジェクトに直接コピー。

### 3. 機能をワークフローに組み合わせる

本当のパワーは機能の組み合わせにある。スラッシュコマンド＋メモリ＋サブエージェント＋フックを連結して、コードレビュー・デプロイ・ドキュメント生成を自動化するパイプラインを構築する方法を学ぶ。

### 4. 理解度をテストする

各モジュールの後に `/lesson-quiz [topic]` を実行する。クイズが何を見落としたかを指摘してくれるので、素早くギャップを埋められる。

## 15分で始める

```bash
# 1. ガイドをクローン
git clone https://github.com/luongnv89/claude-howto.git
cd claude-howto

# 2. 最初のスラッシュコマンドをコピー
mkdir -p /path/to/your-project/.claude/commands
cp 01-slash-commands/optimize.md /path/to/your-project/.claude/commands/

# 3. 試してみる — Claude Code で入力：
# /optimize

# 4. もっと学びたい？プロジェクトメモリをセットアップ：
cp 02-memory/project-CLAUDE.md /path/to/your-project/CLAUDE.md

# 5. スキルをインストール：
cp -r 03-skills/code-review ~/.claude/skills/
```

完全なセットアップをしたいなら、**1時間の必須セットアップ**：

```bash
# スラッシュコマンド（15分）
cp 01-slash-commands/*.md .claude/commands/

# プロジェクトメモリ（15分）
cp 02-memory/project-CLAUDE.md ./CLAUDE.md

# スキルのインストール（15分）
cp -r 03-skills/code-review ~/.claude/skills/

# 週末の目標: フック・サブエージェント・MCP・プラグインを追加
# ガイド付きセットアップは学習パスを参照
```

## FAQ

**無料ですか？**
はい。MIT ライセンスで永久に無料です。個人プロジェクト・職場・チームで使用できます — ライセンス表示を含めることが唯一の条件です。

**メンテナンスされていますか？**
積極的にメンテナンスしています。このガイドはすべての Claude Code リリースに同期されています。現在のバージョン: v2.1.112（2026年4月）、Claude Code 2.1以上に対応。

**公式ドキュメントとどう違うの？**
公式ドキュメントは機能のリファレンスです。このガイドはダイアグラム・本番環境対応テンプレート・段階的な学習パスを備えたチュートリアルです。相互補完的 — まずここで学び、具体的な内容が必要なときは公式ドキュメントを参照してください。

**すべてやるのにどのくらいかかる？**
完全なパスで11〜13時間。でも15分で即座に価値を得られます — スラッシュコマンドのテンプレートをコピーして試してみるだけでOK。

**Claude Sonnet / Haiku / Opus でも使える？**
はい。すべてのテンプレートは Claude Sonnet 4.6、Claude Opus 4.7、Claude Haiku 4.5 で動作します。

**コントリビュートできますか？**
もちろんです。ガイドラインは [CONTRIBUTING.md](CONTRIBUTING.md) を参照してください。新しい例・バグ修正・ドキュメントの改善・コミュニティテンプレートを歓迎します。

**オフラインで読めますか？**
はい。`uv run scripts/build_epub.py --lang ja` を実行して、すべてのコンテンツとレンダリングされたダイアグラムを含む EPUB ebook を生成できます。

<summary>トラブルシューティング</summary>

### 機能が読み込まれない

1. ファイルの場所と名前を確認
2. YAML フロントマターの構文を確認
3. ファイルのパーミッションを確認
4. Claude Code バージョンの互換性を確認

### MCP 接続に失敗する

1. 環境変数を確認
2. MCP サーバーのインストールを確認
3. 認証情報をテスト
4. ネットワーク接続を確認

### サブエージェントが委任されない

1. ツールのパーミッションを確認
2. エージェントの説明の明確さを確認
3. タスクの複雑さを確認
4. エージェントを単独でテスト
