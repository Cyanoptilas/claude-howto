# リファクタリングカタログ

Martin Fowler の *Refactoring*（第2版）からのリファクタリングテクニックのキュレートされたカタログ。各リファクタリングには動機・ステップバイステップのメカニクス・例が含まれています。

> 「リファクタリングはそのメカニクスによって定義される — 変更を実行するために従う正確なステップのシーケンスだ。」— Martin Fowler

---

## このカタログの使い方

1. コードスメルリファレンスを使って**スメルを特定する**
2. このカタログで**対応するリファクタリングを見つける**
3. ステップバイステップで**メカニクスに従う**
4. 振る舞いが保持されていることを確認するために**各ステップ後にテストする**

**ゴールデンルール**: 何かのステップが10分以上かかる場合は、より小さなステップに分解する。

---

## 最も一般的なリファクタリング

### メソッドの抽出

**使う場面**: 長いメソッド・重複コード・コンセプトに名前を付ける必要がある場合

**動機**: コードの断片を、目的を説明する名前のメソッドに変換する。

**メカニクス**:
1. 何をするかに基づいて（どのようにするかではなく）新しいメソッドを作成する
2. コードの断片を新しいメソッドにコピーする
3. 断片で使用されているローカル変数をスキャンする
4. ローカル変数をパラメータとして渡す（またはメソッド内で宣言する）
5. 戻り値を適切に処理する
6. 元の断片を新しいメソッドへの呼び出しに置き換える
7. テストする

---

### 変数名の変更

**使う場面**: 変数の目的が名前から不明確な場合

**動機**: 変数に良い名前を付けることで、その目的が一目で明らかになる。

**メカニクス**:
1. 変数が広い範囲で使われている場合はカプセル化を検討する
2. 変数へのすべての参照を見つける
3. 新しい名前に変更する
4. テストする

---

### パラメータオブジェクトの導入

**使う場面**: よく一緒に使われるデータのグループがある場合

**動機**: 複数のパラメータを持つデータのグループをオブジェクトに変換する。

**メカニクス**:
1. 適切なオブジェクトを作成する
2. メソッドのシグネチャに新しいオブジェクトを追加する
3. 新しいオブジェクトを使用するように関数本体を変更する
4. 古いパラメータを削除する
5. テストする

---

### 条件文をポリモーフィズムに置き換える

**使う場面**: 同じスイッチ文またはif-else チェーンが複数の場所に現れる場合

**動機**: スイッチ文をオブジェクトとポリモーフィズムに置き換えて、コードを拡張しやすくする。

**メカニクス**:
1. 条件文に基づいて継承階層が存在しない場合は作成する
2. 各ケースのロジックをサブクラスのオーバーライドメソッドに移動する
3. 共通コードをスーパークラスに残す
4. 条件文を削除する
5. テストする

---

### クラスの抽出

**使う場面**: クラスが二つのクラスが行う仕事をしている場合

**動機**: 大きなクラスから責任の一部を新しいクラスに分離する。

**メカニクス**:
1. 分割方法を決定する
2. 新しいクラスを作成する
3. 古いクラスから新しいクラスへリンクを作成する
4. 移動したいフィールドを新しいクラスに移動する
5. フィールドを移動するたびにテストする
6. メソッドを移動する（単純なものから始める）
7. インターフェースを確認する
8. テストする

---

### メソッドの移動

**使う場面**: メソッドが自クラスよりも他のクラスの機能を使っている場合

**動機**: メソッドを最もよく使用するクラスに移動する。

**メカニクス**:
1. ソースクラスで使用されるすべての機能を調べる
2. ソースクラスのスーパークラスとサブクラスを確認する
3. ターゲットクラスでメソッドを定義する
4. ソースメソッドからターゲットメソッドにコピーする
5. ターゲットクラスを参照するようにソースメソッドを変更する
6. テストする

---

## クイックリファレンス

| スメル | 推奨リファクタリング |
|-------|---------------------------|
| 長いメソッド | メソッド抽出・テンポラリをクエリに置き換える |
| 重複コード | メソッド抽出・メソッドの引き上げ |
| 巨大なクラス | クラス抽出・サブクラス抽出 |
| フィーチャーエンビー | メソッド移動・フィールド移動 |
| 長いパラメータリスト | パラメータオブジェクトの導入 |
| スイッチ文 | 条件文をポリモーフィズムに置き換える |
| デッドコード | デッドコードの削除 |
| 推測的一般性 | 階層の折り畳み・クラスのインライン化 |
