## ファイルとフォルダーの命名

### レッスンフォルダー

レッスンフォルダーは、**2桁の番号プレフィックス**の後に**ケバブケース**の説明語を使用します:

```
01-slash-commands/
02-memory/
03-skills/
04-subagents/
05-mcp/
```

番号は初心者から上級者への学習パスの順序を反映しています。

### ファイル名

| 種類 | 規約 | 例 |
|------|-----------|----------|
| **レッスン README** | `README.md` | `01-slash-commands/README.md` |
| **機能ファイル** | ケバブケース `.md` | `code-reviewer.md`、`generate-api-docs.md` |
| **シェルスクリプト** | ケバブケース `.sh` | `format-code.sh`、`validate-input.sh` |
| **設定ファイル** | 標準名 | `.mcp.json`、`settings.json` |
| **メモリファイル** | スコープ付きプレフィックス | `project-CLAUDE.md`、`personal-CLAUDE.md` |
| **トップレベルドキュメント** | UPPER_CASE `.md` | `CATALOG.md`、`QUICK_REFERENCE.md`、`CONTRIBUTING.md` |
| **画像アセット** | ケバブケース | `pr-slash-command.png`、`claude-howto-logo.svg` |

### ルール

- すべてのファイル名とフォルダー名には**小文字**を使用する（`README.md`・`CATALOG.md` などのトップレベルドキュメントを除く）
- 単語の区切りには**ハイフン**（`-`）を使用し、アンダースコアやスペースは使わない
- 名前は説明的で簡潔に保つ

## ドキュメント構成

### ルート README

ルート `README.md` は以下の順序に従います:

1. ロゴ（ダーク/ライトバリアントを持つ `<picture>` 要素）
2. H1 タイトル
3. はじめに用ブロッククォート（1行の価値提案）
4. "このガイドを使う理由？" セクション（比較テーブル付き）
5. 水平線（`---`）
6. 目次
7. 機能カタログ
8. クイックナビゲーション
9. 学習パス
10. 機能セクション
11. はじめかた
12. ベストプラクティス / トラブルシューティング
13. コントリビューション / ライセンス

### レッスン README

各レッスンの `README.md` は以下の順序に従います:

1. H1 タイトル（例: `# スラッシュコマンド`）
2. 概要段落
3. クイックリファレンステーブル（オプション）
4. アーキテクチャダイアグラム（Mermaid）
5. 詳細セクション（H2）
6. 実践的な例（番号付き、4〜6例）
7. ベストプラクティス（Do と Don't テーブル）
8. トラブルシューティング
9. 関連ガイド / 公式ドキュメント
10. ドキュメントメタデータフッター

### セクション区切り

水平線（`---`）を使って主要なドキュメント領域を区切ります:

```markdown
---

## 新しい主要セクション
```

## 見出し

### 階層

| レベル | 用途 | 例 |
|-------|-----|---------|
| `#` H1 | ページタイトル（ドキュメントに1つ） | `# スラッシュコマンド` |
| `##` H2 | 主要セクション | `## ベストプラクティス` |
| `###` H3 | サブセクション | `### スキルの追加` |
| `####` H4 | サブサブセクション（まれ） | `#### 設定オプション` |

### ルール

- **H1 はドキュメントに1つ** — ページタイトルのみ
- **レベルをスキップしない** — H2 から H4 にジャンプしない
- **見出しを簡潔に** — 2〜5語を目安に
- **文頭大文字** — 英語では最初の単語と固有名詞のみ大文字（機能名はそのまま）
- **絵文字プレフィックスはルート README のセクションヘッダーのみ**

## テキストのフォーマット

### 強調

| スタイル | 使い場面 | 例 |
|-------|------------|---------|
| **太字** (`**text**`) | 重要な用語、テーブルのラベル、重要なコンセプト | `**インストール**:` |
| *斜体* (`*text*`) | 技術用語の初出、書籍・ドキュメントのタイトル | `*フロントマター*` |
| `コード` (`` `text` ``) | ファイル名、コマンド、設定値、コードの参照 | `` `CLAUDE.md` `` |

### ブロッククォートによるコールアウト

重要なメモには太字プレフィックス付きのブロッククォートを使用します:

```markdown
> **注記**: カスタムスラッシュコマンドは v2.0 からスキルに統合されました。

> **重要**: API キーやクレデンシャルは絶対にコミットしないでください。

> **ヒント**: メモリとスキルを組み合わせると最大の効果が得られます。
```

サポートされているコールアウトタイプ: **注記**、**重要**、**ヒント**、**警告**。

## コードブロック

### 言語タグ

構文ハイライトのために常に言語タグを指定します:

| 言語 | タグ | 用途 |
|----------|-----|---------|
| Shell | `bash` | CLI コマンド、スクリプト |
| Python | `python` | Python コード |
| JavaScript | `javascript` | JS コード |
| TypeScript | `typescript` | TS コード |
| JSON | `json` | 設定ファイル |
| YAML | `yaml` | フロントマター、設定 |
| Markdown | `markdown` | Markdown の例 |
| SQL | `sql` | データベースクエリ |
| プレーンテキスト | （タグなし） | 期待される出力、ディレクトリツリー |

### 規約

- 非明白なコマンドの前には**コメント行**を追加する
- すべての例を**コピー&ペースト対応**にする
- 関連する場合は**シンプルな版と高度な版**の両方を示す

## YAML フロントマター

### 機能ファイル（スキル、コマンド、エージェント）

```yaml
---
name: unique-identifier
description: What this feature does and when to use it
allowed-tools: Bash, Read, Grep
---
```

### ルール

- ファイルの先頭にフロントマターを配置する
- `name` フィールドには**ケバブケース**を使用する
- `description` は1文で収める
- 必要なフィールドのみを含める

## トーンとボイス

### 文章スタイル

- **プロフェッショナルだが親しみやすい** — 専門用語過多にならない技術的な正確さ
- **能動態** — "ファイルを作成する" であり "ファイルが作成される" ではない
- **直接的な指示** — "このコマンドを実行する" であり "このコマンドを実行したいかもしれません" ではない
- **初心者に優しい** — 読者は Claude Code に新しいが、プログラミングには慣れていると仮定する

## コミットメッセージ

[Conventional Commits](https://www.conventionalcommits.org/) に従います:

```
type(scope): description
```

### タイプ

| タイプ | 用途 |
|------|---------|
| `feat` | 新機能、新しい例、新しいガイド |
| `fix` | バグ修正、修正、壊れたリンク |
| `docs` | ドキュメントの改善 |
| `refactor` | 動作を変えない構造の変更 |
| `style` | フォーマットのみの変更 |
| `test` | テストの追加・変更 |
| `chore` | ビルド、依存関係、CI |

### スコープ

レッスン名またはファイル領域をスコープとして使用します:

```
feat(slash-commands): Add API documentation generator
docs(memory): Improve personal preferences example
fix(README): Correct table of contents link
```

## 著者向けチェックリスト

コンテンツを提出する前に確認してください:

- [ ] ファイル/フォルダー名がケバブケースを使用している
- [ ] ドキュメントが H1 タイトルで始まる（ファイルに1つ）
- [ ] 見出し階層が正しい（スキップなし）
- [ ] すべてのコードブロックに言語タグがある
- [ ] コード例がコピー&ペースト対応になっている
- [ ] 内部リンクに相対パスを使用している
- [ ] 外部リンクに説明的なアンカーテキストがある
- [ ] テーブルが適切にフォーマットされている
- [ ] Mermaid ダイアグラムが標準カラーパレットを使用している
- [ ] 機密情報（API キー、クレデンシャル）が含まれていない
- [ ] YAML フロントマターが有効（該当する場合）
- [ ] 画像に alt テキストがある
- [ ] 段落が短くて集中している
- [ ] コミットメッセージが Conventional Commits 形式に従っている
