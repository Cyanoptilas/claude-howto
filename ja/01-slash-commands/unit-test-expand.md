# ユニットテスト拡張

プロジェクトのテストフレームワークに合わせて既存のユニットテストを拡張してください：

1. **カバレッジを分析**: カバレッジレポートを実行して未テストのブランチ・エッジケース・カバレッジが低い箇所を特定
2. **ギャップを特定**: コードの論理分岐・エラーパス・境界条件・null/空の入力をレビュー
3. **プロジェクトのフレームワークを使ってテストを書く**:
   - Jest/Vitest/Mocha（JavaScript/TypeScript）
   - pytest/unittest（Python）
   - Go testing/testify（Go）
   - Rust テストフレームワーク（Rust）
4. **特定のシナリオをターゲットに**:
   - エラー処理と例外
   - 境界値（最小/最大・空・null）
   - エッジケースとコーナーケース
   - 状態遷移と副作用
5. **改善を確認**: カバレッジを再実行して測定可能な増加を確認

新しいテストコードブロックのみを提示してください。既存のテストパターンと命名規則に従ってください。
