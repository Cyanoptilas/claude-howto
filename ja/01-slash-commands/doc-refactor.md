# ドキュメントリファクタリング

プロジェクトの種類に合わせてドキュメント構造をリファクタリングしてください：

1. **プロジェクトを分析**: 種類（ライブラリ/API/ウェブアプリ/CLI/マイクロサービス）・アーキテクチャ・ユーザーペルソナを特定
2. **ドキュメントを集約**: 技術ドキュメントを適切なクロスリファレンス付きで `docs/` に移動
3. **Root README.md**: 概要・クイックスタート・モジュール/コンポーネントの概要・ライセンス・連絡先を含むエントリポイントとして整理
4. **コンポーネントドキュメント**: セットアップとテスト手順を含むモジュール/パッケージ/サービスレベルの README を追加
5. **`docs/` を整理** 関連カテゴリ別に:
   - アーキテクチャ・API リファレンス・データベース・デザイン・トラブルシューティング・デプロイ・コントリビューション（プロジェクトに合わせて調整）
6. **ガイドを作成**（該当するものを選択）:
   - ユーザーガイド: アプリケーションのエンドユーザー向けドキュメント
   - API ドキュメント: エンドポイント・認証・API の例
   - 開発ガイド: セットアップ・テスト・コントリビューションワークフロー
   - デプロイガイド: サービス/アプリの本番デプロイ
7. **Mermaid を使用** すべてのダイアグラム（アーキテクチャ・フロー・スキーマ）

ドキュメントは簡潔で見やすく、プロジェクトの種類に適したものにしてください。
